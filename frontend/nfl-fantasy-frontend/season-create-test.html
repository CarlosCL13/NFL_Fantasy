🎯 User Story Implementado

Como administrador del sistema quiero crear una temporada para entonces definir fechas, semanas y parámetros básicos que soporten las ligas.

🔗 Endpoints del Backend Utilizados

POST /api/seasons
Crea una nueva temporada con sus semanas correspondientes

📝 Estructura de Datos

CreateSeasonDto: - name: string (1-100 caracteres, requerido) - weeksCount: number (1-30 semanas, requerido) - startDate: Date (requerido) - endDate: Date (requerido, debe ser posterior a startDate) - isCurrent: boolean (opcional, default: false)

  • Nombres Autodescriptivos: Variables y métodos con nombres claros

  • Convención de Nombres:

🛡️ Validaciones Implementadas

  • Nombre de temporada: requerido, 1-100 caracteres
  • Número de semanas: requerido, rango 1-30
  • Fechas: requeridas, fecha fin posterior a fecha inicio
